# 双指针刷题笔记

## 模块核心

双指针不是简单地放两个变量，而是利用数组/字符串的结构性减少搜索空间。

常见前提：

- 有序性；
- 单调性；
- 左右边界可收缩；
- 原地写入位置可维护。

写题时先问：

```text
哪个指针负责扫描？
哪个指针负责写入/收缩？
每次移动后，排除掉的搜索空间为什么不可能有答案？
```

## Day 17：去重与三数之和

- 实刷日期：2026-06-25
- 题目：26. 删除有序数组中的重复项；15. 三数之和

### 26. 删除有序数组中的重复项

这是快慢指针/读写指针模型：

- `i` 负责扫描原数组；
- `k` 负责维护下一个写入位置；
- `[0..k-1]` 始终是不重复前缀。

因为数组已经有序，所以只需要判断 `nums[i] != nums[i-1]`。

### 15. 三数之和

三数之和的标准流程：

```text
排序 -> 枚举第一个数 -> 剩下两个数用左右指针夹逼
```

排序之后，左右指针移动有单调依据：

- 当前和太小，`left++` 让和变大；
- 当前和太大，`right--` 让和变小。

去重分两层：

- 固定的 `i` 要去重；
- 找到答案后，`left/right` 都要跳过重复值。

### 今日关键词

- 读写指针；
- 排序后夹逼；
- 去重边界；
- 单调性决定指针移动方向。
