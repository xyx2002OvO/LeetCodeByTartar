# 滑动窗口刷题笔记

## 模块核心

滑动窗口解决的是“连续子数组/子串”问题。核心不是套双指针，而是维护一个窗口不变量：

```text
窗口是否合法？
新增 right 后如何破坏合法性？
移动 left 时如何恢复合法性？
答案在什么时候更新？
```

通用骨架：

```java
for (int right = 0; right < n; right++) {
    add(right);
    while (windowInvalid()) {
        remove(left);
        left++;
    }
    ans = Math.max(ans, right - left + 1);
}
```

## 重要边界：窗口可以为空

滑动窗口收缩时，通常不要额外写：

```java
while (invalid && left < right)
```

除非题目明确不允许空窗口。更稳的原则是：

```text
while 只写业务非法条件。
```

例如 1004「最大连续 1 的个数 III」：

```java
while (zeroCnt > k) {
    if (nums[left] == 0) zeroCnt--;
    left++;
}
```

当 `nums = [0,0,0]` 且 `k = 0` 时，必须允许 `left` 从 `right` 继续走到 `right + 1`，让窗口变成空窗口。否则会把非法窗口 `[right..right]` 误记为答案。

## Day 19：无重复与最多两类

- 实刷日期：2026-06-27
- 题目：3. 无重复字符的最长子串；904. 水果成篮
- 备注：原计划 340/159 是会员题，使用 904 作为非会员替代。904 本质是“最多 2 种元素的最长窗口”。

### 3. 无重复字符的最长子串

窗口不变量：

```text
窗口内每个字符最多出现一次。
```

遇到重复字符时持续移动左边界，直到该字符可以重新加入：

```java
while (exists[c]) {
    exists[s.charAt(left)] = false;
    left++;
}
```

这比专门寻找旧字符位置更统一，因为它直接表达了“不合法就收缩”的窗口思想。

### 904. 水果成篮

窗口不变量：

```text
窗口内不同水果种类 <= 2。
```

用 `Map<水果类型, 出现次数>` 维护窗口状态。右边界扩张后，如果种类超过 2，就移动左边界并扣减计数，直到重新合法。

这题也可以写“上一段连续水果 + 当前水果”的特化版本，但 Map 版是 159/340 的通用母模板。

## Day 22：预算模型

- 实刷日期：2026-06-30
- 题目：424. 替换后的最长重复字符；1004. 最大连续 1 的个数 III

预算模型的共同点：

```text
窗口长度 - 可以保留的数量 <= 可替换预算
```

### 424. 替换后的最长重复字符

固定窗口内，最优策略一定是保留出现次数最多的字符，替换其他字符：

```text
需要替换次数 = windowLen - maxCnt
```

`maxCnt` 可以维护为历史最大频次，不需要在收缩时回退。这是一个单调上界优化：

- 贪心点：固定窗口内保留最高频字符最优；
- 剪枝点：收缩时不重新扫描 26 个字符找当前最高频；
- 正确性：历史 `maxCnt` 偏大，只会让窗口少收缩/晚收缩，不会收缩过头；答案长度也不会虚高到超过历史真实最高频 + k 能支撑的范围。

可以记成：

```text
不漏：maxCnt 偏大 => 收缩更宽松 => 不会过度移动 left。
不虚：maxCnt 来自历史真实频次 => ans 不会超过某个真实频次 + k 的可支撑长度。
```

### 1004. 最大连续 1 的个数 III

这题是更直接的预算模型：

```text
窗口内 0 的数量 <= k
```

0 就是需要翻转的成本。窗口非法时只按 `zeroCnt > k` 收缩，不要额外限制 `left < right`，因为 `k=0` 且当前元素是 0 时，合法窗口可以是空窗口。

## 当前经验总结

- 先定义窗口不变量，再写代码；
- `while` 收缩条件只写业务非法条件，不要随手加边界；
- 最长窗口通常在窗口恢复合法后更新答案；
- 计数型窗口用数组/Map 维护状态；
- 预算型窗口要明确“成本”是什么；
- 有负数的子数组和问题通常不要硬套滑动窗口，优先考虑前缀和。
