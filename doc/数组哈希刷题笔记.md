# 数组哈希刷题笔记

## 模块核心

数组哈希题的核心不是 HashMap API，而是设计映射关系：

```text
key 表示什么？
value 表示什么？
为什么这个 key 能把同类数据归到一起？
```

常见用途：

- 查找补数；
- 记录出现位置；
- 统计频次；
- 分组；
- 前缀状态计数。

## Day 11：索引映射与频次签名

- 实刷日期：2026-06-18
- 题目：1. 两数之和；49. 字母异位词分组

### 1. 两数之和

这题是哈希表的基础模型：

```text
当前值：nums[i]
目标补数：target - nums[i]
```

遍历时先查询补数是否已经出现，再把当前值放入 map。这样可以避免同一个元素被重复使用。

映射设计：

```text
Map<数值, 下标>
```

关键点：

- 先查再放；
- value 存下标，不是存数字本身；
- 如果有重复数字，后出现的下标可以覆盖前面的下标，但本题找到答案后直接返回，影响不大。

### 49. 字母异位词分组

异位词的判断标准是字符频次相同。可以为每个字符串生成一个“频次签名”，把签名相同的字符串放到同一个列表里。

映射设计：

```text
Map<频次签名, 字符串列表>
```

Java 注意点：

```java
String key = Arrays.toString(count);
```

不要写：

```java
count.toString();
```

因为数组的 `toString()` 不是内容字符串，而是对象标识。

### 今日沉淀

这两题分别代表哈希表的两类基本用法：

```text
1. 两数之和：用哈希表做 O(1) 查找。
2. 字母异位词分组：用哈希表做分类聚合。
```

数组哈希模块后续重点是：

- key 的唯一性；
- value 的语义；
- 是否需要处理重复值；
- 是否可以用频次、排序、前缀和、余数等结构作为 key。

## Day 12：TopK 与前缀和

- 实刷日期：2026-06-21
- 题目：347. 前 K 个高频元素；692. 前 K 个高频单词；560. 和为 K 的子数组；523. 连续的子数组和

### 347 / 692：TopK 高频问题

TopK 高频问题通常分两步：

```text
1. HashMap 统计频次
2. 堆 / 桶排序 / 排序 取前 K
```

今天的关键修正是：PriorityQueue 不会因为外部 `Map` 的 value 改变而自动重排。因此应当先完整统计频次，再把元素放入堆。

347 只需要按频次排序；692 还需要处理频次相同时的字典序：

```text
频次高的在前
频次相同，字典序小的在前
```

### 560：和为 K 的子数组

这题是前缀和计数：

```text
prefix[j] - prefix[i] = k
=> prefix[i] = prefix[j] - k
```

因此遍历到当前前缀和 `prefix` 时，需要知道之前出现过多少次 `prefix - k`。

由于数组里可能有负数，不能用滑动窗口。

推荐顺序是：

```text
先 ans += count(prefix - k)
再 count(prefix)++
```

这样不需要对 `k == 0` 做额外修正。

### 523：连续的子数组和

这题是前缀和余数：

```text
(prefix[j] - prefix[i]) % k == 0
=> prefix[j] % k == prefix[i] % k
```

所以只要两个前缀和对 k 的余数相同，中间这段子数组的和就可以被 k 整除。

关键点：

- `mp.put(0, -1)` 用于处理从下标 0 开始的子数组；
- Map 中保存某个余数第一次出现的位置；
- 必须检查子数组长度至少为 2。

### 今日关键词

- TopK：先统计，再建堆；
- PriorityQueue 不会监听外部 Map 变化；
- 560：前缀和出现次数；
- 523：前缀和同余 + 最早下标；
- 有负数时优先考虑前缀和，不要套滑动窗口。

## Day 14：二维前缀和与差分数组

- 实刷日期：2026-06-22
- 题目：304. 二维区域和检索 - 矩阵不可变；1109. 航班预订统计

### 304. 二维区域和检索 - 矩阵不可变

二维前缀和的核心是用一个额外的哨兵行列，把坐标边界统一掉：

```text
prefix[i + 1][j + 1]
= prefix[i][j + 1] + prefix[i + 1][j] - prefix[i][j] + matrix[i][j]
```

查询子矩阵 `[row1..row2] x [col1..col2]` 时：

```text
sum = prefix[row2 + 1][col2 + 1]
    - prefix[row1][col2 + 1]
    - prefix[row2 + 1][col1]
    + prefix[row1][col1]
```

这里最容易混的是 `+1` 的位置：

- 右下角要完整覆盖查询区域，所以用 `row2 + 1` 和 `col2 + 1`；
- 左上角表示被排除区域的边界，直接用 `row1` 和 `col1`。

### 1109. 航班预订统计

这题是差分数组的标准区间加法：

```text
对区间 [first, last] 加 seats
diff[first - 1] += seats
diff[last] -= seats   // 前提是 last < n
```

最后对 `diff` 做一次前缀和，就得到每个航班的最终座位数。

注意点：

- 题目里的航班编号是 1-based，数组下标是 0-based；
- `last == n` 时不能写 `diff[last]`，否则越界；
- 差分数组适合“多次区间修改，最后统一查询每个点”的模型。

### 今日关键词

- 304：二维前缀和 + 哨兵行列；
- 1109：差分数组 + 前缀和还原；
- 前缀和是“快速查区间和”，差分是“快速做区间加法”。


## Day 15：原地标记（循环置换）

- 实刷日期：2026-06-23
- 题目：41. 缺失的第一个正数；448. 找到所有数组中消失的数字

### 模块共性

这两题是同一类母题：

```text
值域在 [1..n]，把值 x 放到下标 x-1 的位置上。
```

落位完成后再统一扫描：

- 第一个 `nums[i] != i+1` 对应“最小缺失正数”；
- 所有 `nums[i] != i+1` 对应“所有缺失数字”。

### 关键编码点

- 当前位置要用 `while` 连续交换，不是 `if`；
- 交换前要防重：`nums[nums[i]-1] != nums[i]`，避免死循环；
- 41 题需要先做范围过滤（`<=0` 或 `>n` 无需落位）；
- 448 题因题目保证值域在 `[1..n]`，可直接落位。

### 今日关键词

- 原地哈希（index as bucket）
- 循环置换（cyclic sort）
- 一次落位 + 一次扫描


## Day 16：三路划分与虚拟下标

- 实刷日期：2026-06-24
- 题目：75. 颜色分类；324. 摆动排序 II

### 75. 颜色分类

这是三路划分模板题：

- 左区间放 0
- 中区间放 1
- 右区间放 2

核心细节是：和右侧交换后，当前位置换回来的值还没分类，因此需要继续处理。

### 324. 摆动排序 II

这题是“快选 + 三路划分 + 虚拟下标映射”的组合。

- 快选负责找到中位数；
- 三路划分负责把值域分成 `>mid / ==mid / <mid`；
- 虚拟下标负责把大值优先放到奇数位，满足摆动不等式。

映射公式：

```text
idx(i) = (2 * i + 1) % (n | 1)
```

### 今日关键词

- DNF（三色旗）
- 快选找中位数
- 虚拟下标映射
- 值语义分区 + 位置语义映射
