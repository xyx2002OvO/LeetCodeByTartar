# 数组哈希刷题笔记

## 模块核心

数组哈希题的核心不是 HashMap API，而是设计映射关系：

```text
key 表示什么？
value 表示什么？
为什么这个 key 能把同类数据归到一起？
```

常见用途：

- 查找补数；
- 记录出现位置；
- 统计频次；
- 分组；
- 前缀状态计数。

## Day 11：索引映射与频次签名

- 实刷日期：2026-06-18
- 题目：1. 两数之和；49. 字母异位词分组

### 1. 两数之和

这题是哈希表的基础模型：

```text
当前值：nums[i]
目标补数：target - nums[i]
```

遍历时先查询补数是否已经出现，再把当前值放入 map。这样可以避免同一个元素被重复使用。

映射设计：

```text
Map<数值, 下标>
```

关键点：

- 先查再放；
- value 存下标，不是存数字本身；
- 如果有重复数字，后出现的下标可以覆盖前面的下标，但本题找到答案后直接返回，影响不大。

### 49. 字母异位词分组

异位词的判断标准是字符频次相同。可以为每个字符串生成一个“频次签名”，把签名相同的字符串放到同一个列表里。

映射设计：

```text
Map<频次签名, 字符串列表>
```

Java 注意点：

```java
String key = Arrays.toString(count);
```

不要写：

```java
count.toString();
```

因为数组的 `toString()` 不是内容字符串，而是对象标识。

### 今日沉淀

这两题分别代表哈希表的两类基本用法：

```text
1. 两数之和：用哈希表做 O(1) 查找。
2. 字母异位词分组：用哈希表做分类聚合。
```

数组哈希模块后续重点是：

- key 的唯一性；
- value 的语义；
- 是否需要处理重复值；
- 是否可以用频次、排序、前缀和、余数等结构作为 key。
