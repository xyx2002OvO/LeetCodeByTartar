# LeetCode 21 你的版本（v2）优缺点

## 代码对象

当前分析对象是你在 `/Users/bytedance/.leetcode/21.合并两个有序链表.java` 里的 version 2 思路：

- 循环条件：`while (list1 != null || list2 != null)`
- 当某个链表为空时：`cur.next = 另一个链表; break;`
- 其余情况：逐节点比较并推进

## 优点

1. **思路正确，能 AC**
   - 核心逻辑没有偏差，比较两个有序链表头节点后链接较小者，最后处理剩余链表。

2. **使用了 dummy 节点，结构清晰**
   - `dummy + cur` 的模式减少了头节点特判，属于链表题的推荐写法。

3. **相比 v1 明显优化**
   - v1 在某个链表为空后仍按“每轮只接一个节点”的方式推进（`continue`），
     v2 改成直接挂剩余链表并 `break`，减少了冗余循环。

4. **边界覆盖完整**
   - `list1` 或 `list2` 一开始就为 `null` 的情况可正确处理。

## 缺点

1. **循环条件与循环体职责不够聚焦**
   - `while (list1 != null || list2 != null)` 包含了“任一非空”场景，
     但核心比较逻辑只在“两者都非空”时成立，导致循环体内部需要额外空指针分支。

2. **分支复杂度略高于标准写法**
   - 每轮都要先判断 `list1 == null / list2 == null`，然后才进入比较分支。
   - 可读性上不如“主循环只处理公共逻辑，尾部一次收尾”的结构。

3. **文件中保留了两个 `class Solution`（提交风险）**
   - 这是当前文件状态问题，不是算法问题。
   - LeetCode 提交时只能保留一个 `Solution` 类，否则会编译失败。

4. **代码风格可进一步统一**
   - 运算符和关键字周围空格偏紧（如 `list1!=null`），
     对复杂题后期维护不利。

## 结论

- 你的 v2 已经是**正确且实用**的版本，尤其较 v1 有实质改进。
- 若追求“面试官一眼可读”的风格，建议再演进到标准题解结构（`&&` 主循环 + 尾部挂链）。
