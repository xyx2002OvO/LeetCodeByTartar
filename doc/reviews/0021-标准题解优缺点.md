# LeetCode 21 标准题解优缺点

## 参考结构（主流）

```java
class Solution {
    public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
        ListNode dummy = new ListNode(0);
        ListNode cur = dummy;

        while (list1 != null && list2 != null) {
            if (list1.val <= list2.val) {
                cur.next = list1;
                list1 = list1.next;
            } else {
                cur.next = list2;
                list2 = list2.next;
            }
            cur = cur.next;
        }

        cur.next = (list1 != null) ? list1 : list2;
        return dummy.next;
    }
}
```

## 优点

1. **结构职责清晰**
   - 主循环只处理“两者都非空”的公共比较逻辑：`while (list1 != null && list2 != null)`。
   - 收尾逻辑统一放在循环外，一次挂接剩余链表。

2. **分支少、可读性高**
   - 循环内无需 `list1 == null / list2 == null` 的额外判断，
     对调试和 code review 更友好。

3. **模板复用性强**
   - 这是很多链表双指针题的标准骨架，易迁移到“k 路归并”等更复杂场景。

4. **时间与空间复杂度最优**
   - 时间复杂度 `O(m + n)`，空间复杂度 `O(1)`（不计输出链表）。

## 缺点（或注意点）

1. **对初学者理解门槛略高**
   - 初学时可能不如“全在循环内判断”的写法直观。

2. **过度模板化可能导致机械套用**
   - 若不理解不变量（`cur` 总在结果链尾），容易在其他题型中误用。

3. **收尾语句依赖前置正确性**
   - `cur.next = (list1 != null) ? list1 : list2;` 简洁但要求前面循环推进完全正确。

## 适用建议

- 面试、复盘、长期维护：优先使用标准写法。
- 训练期从“能 AC”过渡到“高可读”：可以先写你的 v2，再重构成标准写法。
