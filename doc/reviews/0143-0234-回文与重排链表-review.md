# LeetCode 234 / 143 回文与重排链表 Review

## 今日主题

这两题共用同一个链表组合套路：

```text
快慢指针找中点 -> 断开链表 -> 反转后半段 -> 双指针处理两段链表
```

区别在于最后一步：

- 234 是比较前半段和反转后的后半段；
- 143 是把前半段和反转后的后半段交错合并。

## 234. 回文链表

### 代码评价

代码整体思路正确：

1. 使用 dummy + fast/slow 找到中点附近；
2. `slow.next = null` 断开前后两段；
3. 反转后半段；
4. 前半段和反转后的后半段逐节点比较。

优点：

- 对空链表和单节点链表做了提前返回；
- 主流程清晰，符合链表回文题的标准解法；
- 使用反转后半段避免了额外数组或栈，空间更优。

需要注意：

- 当前写法会改变原链表结构，题目通常允许；如果面试官要求不破坏链表，需要比较结束后再把后半段反转回来并接回去。
- 找中点时从 dummy 出发，会让 `slow` 停在前半段尾部，这个写法可以，但要明确奇偶长度下 `mid = slow.next` 的含义。
- 234 文件只记录了 beginTime，没有记录 endTime；后续建议两题都记录完整时间。

## 143. 重排链表

### 代码评价

代码整体正确，使用了和 234 相同的前置流程：

1. 找中点；
2. 断开；
3. 反转后半段；
4. 把后半段节点逐个插入前半段。

优点：

- 复用了反转后半段的思路；
- 交错合并时保存了 `mid.next`，避免丢失后半段剩余节点；
- `slow = slow.next.next` 能跳到下一轮前半段插入位置，逻辑成立。

需要注意：

- 交错合并是这题最容易断链的地方，建议以后固定写清楚三个临时变量：

```java
ListNode next1 = first.next;
ListNode next2 = second.next;
first.next = second;
second.next = next1;
first = next1;
second = next2;
```

- 你当前写法更像“把反转后的后半段逐个头插到 slow 后面”，能过，但面试表达时建议说成“两段链表交错合并”，更容易让人理解。

## 今日沉淀

这两题说明链表题经常不是单一技巧，而是多个基础操作组合：

```text
找中点 + 断链 + 反转 + 比较/合并
```

后续遇到复杂链表题，可以先拆成这些基础动作，而不是直接写完整逻辑。
