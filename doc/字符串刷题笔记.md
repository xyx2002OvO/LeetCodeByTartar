# 字符串刷题笔记

## 模块核心

字符串题通常不是单纯操作字符，而是在维护一种结构：

- 回文结构；
- 括号/表达式结构；
- 匹配结构；
- 频次结构；
- 前后缀结构。

做题前先判断：

```text
这题是在问子串、子序列，还是模式匹配？
状态是否依赖区间？
是否需要枚举中心、边界、前缀或栈状态？
```

## Day 23：回文字符串

- 实刷日期：2026-07-01
- 题目：5. 最长回文子串；647. 回文子串

### 共同 DP 状态

两题都可以用同一个区间 DP 状态：

```text
dp[i][j] 表示 s[i..j] 是否是回文串。
```

转移：

```text
s[i] == s[j] 且 (j - i < 3 或 dp[i+1][j-1])
```

其中 `j - i < 3` 覆盖长度 1、2、3 的短区间：

- 长度 1：天然回文；
- 长度 2：两端相等即可；
- 长度 3：两端相等，中间一个字符天然回文。

遍历顺序需要保证内部区间先被计算。按右端点 `j` 从小到大，左端点 `i <= j` 遍历即可。

### 5. 最长回文子串

在每次发现 `dp[i][j] == true` 时，用当前区间长度更新最长答案。

### 647. 回文子串

在每次发现 `dp[i][j] == true` 时，答案计数加一。

### 今日关键词

- 区间 DP；
- 短区间 base case；
- 右端点驱动遍历；
- 同一状态，不同输出目标。

## 经验补充

回文子串类题还有一个常用解法：中心扩展。中心扩展空间更省，DP 更容易和后续区间 DP 思维衔接。

面试时可以按需求选择：

- 要快速写出稳定版本：中心扩展；
- 要训练区间状态转移：DP；
- 后续遇到回文子序列/切割问题：优先考虑 DP。
