# 刷题笔记

## 链表模块

### 数字加法类链表题

这类题的第一步不是写代码，而是先判断数字在链表中的存储顺序：

- **低位在前**：可以直接从头节点开始逐位相加，例如 LeetCode 2。
- **高位在前**：需要先把问题转成低位优先的形式，例如反转链表或使用栈，例如 LeetCode 445。

通用加法框架是：同步遍历两个链表，用 `carry` 保存进位，用 dummy 节点构造结果链表。循环条件要覆盖三个情况：`l1` 未结束、`l2` 未结束、`carry` 未清零。

### 链表编码风格

- 构造新链表时优先使用 `dummy + cur`，减少头节点特判。
- 进位变量命名为 `carry`，当前位总和命名为 `sum`，不要用 `next` 表示进位。
- `next` 更适合保留给链表临时指针，例如 `ListNode next = cur.next`。
- 辅助方法尽量写成 `private`，并补齐空链表等通用边界。
- 链表题的核心是维护指针语义：每个指针代表什么、移动后不变量是否还成立。

### 反转链表的使用场景

反转链表不是目的，而是一种“改变处理顺序”的手段。遇到需要从尾部开始处理、但链表只能从头遍历的问题，可以考虑：

1. 反转链表，直接改变遍历方向；
2. 使用栈，保留原链表结构；
3. 递归回溯，利用调用栈从后往前处理。

是否能修改输入链表，会影响方案选择。
